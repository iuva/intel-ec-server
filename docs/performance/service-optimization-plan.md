# 服务性能优化方案 - 700并发瓶颈诊断与优化

## 🔍 问题分析

### 当前问题

- 大量请求超时（30秒超时）
- 即使优化了连接池配置，仍然有性能问题
- 需要找出真正的瓶颈点

### 潜在瓶颈点

#### 1. 数据库连接阻塞 ⚠️ 严重

**问题**：

- `query_available_hosts` 方法中存在循环查询数据库
- 每次循环都可能创建新的数据库会话
- 长时间持有数据库连接，导致连接池耗尽

**代码位置**：

```python
# services/host-service/app/services/host_discovery_service.py
for iteration in range(max_iterations):  # 最多100次循环
    # 调用外部接口
    hardware_hosts = await self._fetch_hardware_hosts(...)
    # 查询数据库（每次循环都查询）
    available_hosts = await self._filter_available_hosts(hardware_ids)
```

**影响**：

- 单个请求可能占用数据库连接数分钟
- 700并发 × 每个请求占用1个连接 = 需要700+连接
- 但连接池只有700，没有余量

#### 2. 外部接口调用阻塞 ⚠️ 中等

**问题**：

- 循环中串行调用外部硬件接口
- 每次调用超时10秒
- 如果外部接口慢，会累积延迟

#### 3. 数据库查询效率 ⚠️ 中等

**问题**：

- 循环中多次查询数据库
- 每次查询可能扫描大量数据
- 缺少查询结果缓存

#### 4. 连接池监控缺失 ⚠️ 中等

**问题**：

- 无法实时查看连接池使用情况
- 无法诊断连接阻塞问题
- 缺少连接等待时间监控

## 🔧 优化方案

### 方案 1: 添加连接池监控日志（立即执行）✅

**目标**：实时监控连接池状态，诊断连接阻塞

**实现**：

1. 在数据库管理器中添加连接池状态监控
2. 定期记录连接池使用情况
3. 记录连接获取等待时间
4. 记录连接池耗尽事件

**预期效果**：

- 能够实时查看连接池使用率
- 能够识别连接阻塞的请求
- 能够诊断连接池耗尽问题

### 方案 2: 优化数据库查询逻辑（立即执行）✅

**目标**：减少数据库连接占用时间

**优化点**：

1. **批量查询优化**：减少循环中的数据库查询次数
2. **连接复用**：在循环中复用同一个数据库会话
3. **查询结果缓存**：缓存外部接口数据，减少重复查询

**实现**：

```python
# 优化前：每次循环都创建新会话
for iteration in range(max_iterations):
    available_hosts = await self._filter_available_hosts(hardware_ids)

# 优化后：复用同一个会话
session_factory = mariadb_manager.get_session()
async with session_factory() as session:
    for iteration in range(max_iterations):
        available_hosts = await self._filter_available_hosts_in_session(session, hardware_ids)
```

### 方案 3: 优化外部接口调用（短期优化）📋

**目标**：减少外部接口调用时间

**优化点**：

1. 增加外部接口超时时间
2. 使用并发调用（如果可能）
3. 添加外部接口调用缓存

### 方案 4: 数据库查询优化（中期优化）📋

**目标**：提高数据库查询效率

**优化点**：

1. 检查并优化数据库索引
2. 优化查询语句
3. 使用 Redis 缓存查询结果

## 📊 连接池监控日志设计

### 监控指标

1. **连接池状态**
   - 当前连接数（active）
   - 空闲连接数（idle）
   - 最大连接数（max）
   - 连接池使用率（usage %）

2. **连接获取性能**
   - 连接获取等待时间
   - 连接获取超时次数
   - 连接池耗尽次数

3. **连接生命周期**
   - 连接创建时间
   - 连接使用时长
   - 连接回收时间

### 日志格式

```json
{
  "timestamp": "2025-12-04T21:30:00",
  "level": "INFO",
  "message": "数据库连接池状态",
  "pool_size": 200,
  "max_overflow": 500,
  "active_connections": 650,
  "idle_connections": 50,
  "total_connections": 700,
  "usage_percent": 92.86,
  "waiting_requests": 10,
  "pool_exhausted": false
}
```

### 告警阈值

- **连接池使用率 > 80%**：警告
- **连接池使用率 > 95%**：严重警告
- **连接池耗尽**：立即告警
- **连接获取等待时间 > 5秒**：警告
